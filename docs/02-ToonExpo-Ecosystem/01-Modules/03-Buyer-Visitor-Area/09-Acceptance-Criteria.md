# Acceptance Criteria

## Profile / QR

- Buyer can access profile.
- Buyer can see My QR.
- Unregistered visitor has no QR.
- My QR works for builder scan and entrance check-in contexts.

## Favorites / Saved

- Buyer can save project.
- Buyer can save apartment.
- Buyer can view favorites/saved apartments.
- Buyer can remove saved item.
- Favorites require login.

## Requests / History

- Buyer can send request from project/apartment page.
- Buyer can see sent requests.
- Buyer can see builder-created QR interaction if builder saved/created CRM request.
- Buyer does not see internal CRM notes.
- Buyer does not see other buyers' data.

## Check-in

- Buyer can see current/latest check-in status if available.
- Check-in status is not payment/ticket wallet.

## Privacy

- Buyer sees only own personal area data.
- Entrance Staff sees only check-in result.
- Builder sees buyer data only through legitimate request/QR interaction.
