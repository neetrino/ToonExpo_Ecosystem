# Privacy And Visibility

## Buyer Privacy Rule

Buyer can see only own:

- profile;
- QR;
- favorites;
- requests;
- history;
- check-in records.

## Builder Visibility

Builder sees buyer data only after legitimate interaction:

- buyer sends request;
- builder scans buyer QR and saves/creates request;
- builder has CRM record linked to buyer.

## Partner Visibility

Partners do not see buyer personal data in v1 unless a future partner request flow is explicitly added.

## Entrance Staff Visibility

Entrance Staff sees minimal check-in data only.

No request history, favorites or CRM data.

## BigProjects Admin Visibility

BigProjects Admin can view/manage buyer data when needed for support/platform operations.

Access should be audited for sensitive actions if implemented.

## Internal Notes

Buyer must not see:

- builder CRM notes;
- readiness internal notes;
- admin audit log.
