# Global Entity Management

## Purpose

BigProjects Admin must be able to manage all core ToonExpo entities.

This is important because BigProjects may initially create and maintain data for builders and partners.

## Managed Entity Areas

BigProjects Admin can manage:

- companies/accounts;
- builder profiles;
- projects/buildings/floors/apartments;
- media;
- visual maps/hotspots;
- partners;
- bank offers;
- service providers;
- readiness categories/assessments;
- exhibition maps/booths/check-in summary;
- settings/languages.

## Builder Data

BigProjects Admin can create/edit:

- builder company;
- project hierarchy;
- apartment inventory;
- public media;
- visual map/hotspots.

## Partner Data

BigProjects Admin can create/edit:

- partner company;
- partner type;
- partner public profile;
- bank offers for bank partners.

## Service Providers

BigProjects Admin can create/edit:

- provider records;
- provider categories;
- readiness category links.

## CRM Boundary

BigProjects Admin can view CRM analytics/support info.

Do not allow default editing of builder CRM sales pipeline in v1.

## Bulk / Import

v1 can start manual.

Potential later helpers:

- apartment spreadsheet import;
- booth spreadsheet import;
- media bulk upload;
- duplicate project/floor structures.
