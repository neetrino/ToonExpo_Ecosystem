# Acceptance Criteria

## Admin Access

- BigProjects Admin can access admin dashboard.
- BigProjects Admin can manage all public/platform entities.
- BigProjects Admin can edit builder/project/apartment content.
- BigProjects Admin can edit partner/bank/service provider content.
- BigProjects Admin can manage event map/booth content.

## Publication

- Public entities support draft/published/archived.
- Draft entities are not visible publicly.
- Published entities are visible publicly.
- Archived entities are hidden but retained.
- Publish/archive actions are audited.

## Settings / Languages

- Admin can view platform settings.
- Important settings changes are audited.
- Public content can support Armenian/Russian/English translations.

## Boundaries

- Admin module does not edit builder CRM sales data by default.
- Admin module does not replace BOS.
- Admin module does not own readiness internal notes outside Readiness.
- Admin module does not introduce heavy moderation queue in v1.
