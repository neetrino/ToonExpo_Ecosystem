# Mobile App-Like UX

## Purpose

Mobile should feel like an app even when delivered as web/mobile web.

## Main Mobile Areas

- project browsing;
- apartment browsing;
- favorites;
- requests;
- My QR;
- Public Exhibition Map;
- builder QR scan action if builder user;
- search/filter.

## Navigation

Recommended mobile navigation:

- bottom navigation for key buyer areas;
- prominent My QR access;
- search access;
- back navigation within project/building/floor/apartment hierarchy;
- event map shortcut during active event.

## Mobile Rules

- large tap targets;
- readable apartment pages;
- sticky primary CTA on apartment page;
- fast image loading;
- no desktop-only tables for critical flows;
- visual maps must have list fallback;
- QR screen must be easy to open.

## App Publication Path

App can be published through Neetrino App Technology when web/mobile web is ready.

Product rule:

```text
If web/mobile experience is ready, app shell should be mostly ready too.
```

## Out Of Current Production Scope

- push notifications;
- complex offline mode;
- paid ticket wallet;
- native-only features.
