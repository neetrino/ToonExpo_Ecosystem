# Event And Check-in Analytics

## Purpose

Event analytics helps BigProjects understand event activity.

## Check-in Metrics

Show:

- total check-ins;
- check-ins by hour/day;
- duplicate check-in attempts;
- invalid QR scans;
- blocked QR scans;
- entrance staff scan counts if useful.

## Map Metrics

Show:

- booth selections;
- searches;
- route requests;
- top searched companies/projects;
- top selected booths.

## QR Metrics

Show:

- builder QR scans;
- entrance check-in scans;
- invalid scans;
- blocked scans.

## Boundary

Check-in analytics does not equal ticket/payment reporting.

There are no paid tickets in v1.

## Event Context

Analytics should support event filter:

- current event;
- past event;
- all time.
