# Definition And Boundaries

## Definition

Analytics collects and summarizes important product activity.

It answers:

- what is being viewed?
- what is being requested?
- which builders/projects are active?
- how many QR scans/check-ins happened?
- how ready are builders/projects?
- how is CRM pipeline performing?

## What Analytics Owns

Analytics owns:

- event tracking model;
- metric definitions;
- dashboard summaries;
- filtered views by role;
- basic exports if needed.

## What Analytics Does Not Own

Analytics does not own:

- CRM workflow;
- readiness scoring logic;
- check-in validation logic;
- QR token logic;
- payment/ticket reports;
- full data warehouse in v1.

## Audience

BigProjects Admin:

- global platform/event analytics.

Builder:

- own company/project/CRM analytics.

Partner:

- own profile/offer analytics if enabled.

Entrance Staff:

- check-in only.

Buyer:

- no analytics dashboard in v1.

## v1 Rule

Use practical aggregate metrics first.

Do not block v1 on advanced attribution, funnels, cohort analysis or BI tooling.
