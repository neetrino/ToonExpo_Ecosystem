# Acceptance Criteria

## Access

- [ ] Builder Portal is available only to approved/provisioned builder users and BigProjects Admin.
- [ ] Builder users cannot self-register into Builder Portal.
- [ ] Builder users can see only their own company workspace.
- [ ] BigProjects Admin can select/manage any builder company.

## Portal Shell

- [ ] Builder has a clear dashboard/home after login.
- [ ] Main sections are visible and understandable.
- [ ] Empty states explain what is missing and where to continue.
- [ ] Mobile layout is usable for dashboard, CRM, readiness and basic profile review.

## Company And Team

- [ ] Company profile can be created/edited.
- [ ] Public fields and internal fields are separated.
- [ ] Company team/members can be viewed.
- [ ] Simple v1 access model works without complex builder subroles.

## Projects And Inventory

- [ ] Builder/Admin can manage projects, buildings, floors and apartments through the portal.
- [ ] Apartment inventory shows available/reserved/sold status.
- [ ] Apartment details can link to CRM deals when relevant.
- [ ] Media is attached to real entities, not managed through a separate generic files module.

## UX Pattern

- [ ] Entity clicks open side sheets over the current workspace.
- [ ] Related entities can open stacked sheets.
- [ ] Full pages are used only for real workspaces.
- [ ] Quick dialogs are used only for short creation or confirmation flows.
- [ ] CRM board/list follows the shared operational UI pattern.

## Cross Module Flow

- [ ] Public buyer requests appear in Constructor CRM.
- [ ] Builder QR scan actions appear in the same Constructor CRM flow.
- [ ] Readiness weak points can link to Service Provider Directory categories.
- [ ] Visual Map setup is reachable from project/building/floor/apartment context.
- [ ] Exhibition map booth details can link to the builder/company public presence.

## Admin Support

- [ ] BigProjects Admin can fill builder data during early operation.
- [ ] Admin changes to important data are auditable.
- [ ] Publication controls prevent drafts/internal data from appearing publicly.
