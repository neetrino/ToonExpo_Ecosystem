# ToonExpo Mobile UX

## Purpose

ToonExpo should be mobile-first. The mobile web/app experience is a core product surface.

## Main Mobile Areas

- project browsing;
- apartment browsing;
- favorites;
- buyer QR;
- QR scanner;
- requests;
- public exhibition map;
- participant/project/area/landmark search;
- destination highlight and nearby orientation landmarks;
- optional manual approximate `I am here` marker;
- builder scan buyer QR action page.

## UX Rules

- app-like navigation;
- large tap targets;
- fast access to QR;
- fast access to favorites;
- search/filter should be mobile-friendly;
- apartment pages should be readable on phone;
- visual map/hotspots must work on mobile.
- public exhibition map must support pan/zoom, full-area selection, detail sheet and list fallback.

## Out Of Current Production Scope

- push notifications;
- complex offline mode;
- paid ticket wallet.
- professional indoor routing and automatic positioning.
