# Module Spec Template

Use this template for every product module.

## 1. Purpose

Shortly explain why this module exists.

## 2. Users

List the user roles that use this module.

## 3. In Scope

List what belongs to this module in v1.

## 4. Out Of Scope

List what should not be built here.

## 5. Main Flows

Describe the core user/business flows.

## 6. Pages / Screens

List the required UI pages or screens.

## 7. Data Entities

List the key entities, fields and relationships.

## 8. Permissions

List who can view/create/edit/delete/approve.

## 9. Statuses

List lifecycle statuses and what each means.

## 10. Integrations

Describe module dependencies and external/internal integrations.

## 11. Acceptance Criteria

Define how we know the module is correctly implemented.

## 12. Later

List future improvements that should not block v1.
